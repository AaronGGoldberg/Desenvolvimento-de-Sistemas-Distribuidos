    Decisões técnicas adotadas

    • Identificação: o servidor atribui IDs sequenciais, como cliente-1, e aceita um nome amigável enviado pela interface.
    • Armazenamento: as conexões ativas ficam em um Map, usando o ID do cliente como chave.
    • Desconexão: ao fechar a aba ou perder a conexão, o servidor remove o cliente do Map e avisa os demais.
    • Formato: as mensagens usam JSON para transportar tipo, origem, texto e data/hora de forma estruturada.
